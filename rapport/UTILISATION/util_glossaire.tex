\chapter{Glossaire}

\begin{itemize}

\item[$\bullet$]{\bf CSV} : Coma Separated Value, ou CSV, est un format de données. Un fichier de ce type contient des valeurs dans une table représentée par une série de lignes de caractères organisée de telle manière que chaque colonne est séparée de la colonne suivante par un point-virgule (ou une virgule, ou une tabulation...) et que chaque retour chariot désigne la fin d'une ligne.\\

\item[$\bullet$] {\bf DIP} : Database of Interacting Proteins, ou DIP désigne une base de données d'interactions protéine-protéine.\\

\item[$\bullet$]{\bf Processeur XSLT} : Le processeur XSLT, composant logiciel chargé de la transformation du fichier XML, crée un structure logique arborescente à partir du document XML et lui fait subir des transformations selon les \texttt{templates} contenus dans la feuille XSL pour produire un arbre résultat représentant la structure d'un nouveau document au format XML, CSV, HTML...\\ 
\item[$\bullet$] {\bf PSI} : Proteomic Standard Initiative, ou PSI, désigne un format de données. Ce format a été développé afin de standardiser les format de données relatives aux interactions protéine-protéine. Il existe deux variantes de ce format nommées PSI canonique et PSI d\'eroul\'e.\\

\item[$\bullet$]{\bf Template} : Chaque \texttt{template} définit des traitements à effectuer sur un élément (noeud ou feuille) de l'arborescence XML.\\

\item[$\bullet$]{\bf XIN} : Le format XIN est un format XML très flexible qui permet de décrire
des graphes. La base de donnée DIP est une application particulière de
ce format.\\

\item[$\bullet$]{\bf XSD} : XML Schema Definition. Fichier XML donnant la définition d'un format de fichier XML\\

\item[$\bullet$]{\bf XML} : eXtensible Markup Language, ou XML, est un méta-langage extensible dérivé de SGML permettant de structurer des données.\\

\item[$\bullet$]{\bf XSLT/feuille de style} : eXtensible Stylesheet Language Transformation, ou XSLT, est un langage permettant de transformer la structure des fichiers au format XML. Ainsi, un document XML peut être représenté comme une structure arborescente. XSLT permet alors de transformer les documents XML à l'aide de feuilles de style contenant des règles appelées \texttt{template}.\\






\end{itemize}
